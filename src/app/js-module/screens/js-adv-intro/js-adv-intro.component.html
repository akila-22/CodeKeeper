  • JavaScript code call stack la dha execute aagum
  • The process starts by compiling the code and creating a global execution context for top-level code (code not inside any function).
  • An execution context is an environment where a piece of JavaScript is executed. It stores necessary information for the code to be executed such as local variables or arguments passed into a function.
  • In any JavaScript project, there is only ever one global execution context. It's always there as the default context and it's where top-level code will execute.
  • For each function call, a new execution context will be created containing all the necessary information to run that function. All these execution contexts together make up the call stack.
  • Inside each execution context is a variable environment where all variables and function declarations are stored.
  • A function can also access variables outside of the function due to the scope chain. The scope chain consists of references to variables located outside the current function and it is stored in each execution context.
  • Execution contexts belonging to arrow functions do not get their own arguments keyword, nor do they get the this keyword. They can use the arguments object, and the this keyword from their closest regular function parent.
  • The call stack is a place where execution contexts get stacked on top of each other, to keep track of where we are in the program's execution. The execution context at the top of the stack is the one currently running.
  • A JavaScript engine is a program that executes JavaScript code.
  • JavaScript engine components include a call stack, where code is executed, and a heap, an unstructured memory pool storing all objects needed by our application.
  • JavaScript code is compiled into machine code for execution through a process called just-in-time compilation.
  • The process includes parsing the code into a data structure known as the abstract syntax tree (AST), then compiling it into machine code, and finally executing it.
  • An optimisation strategy is used where an unoptimised version of machine code is created first, then optimized and recompiled during program execution.
  • A JavaScript runtime is a container that includes all necessary components to use JavaScript; at the heart of it is a JavaScript engine.
  • A JavaScript runtime also includes access to web APIs (outside of the JavaScript language itself) and a callback queue for executing callback functions.
